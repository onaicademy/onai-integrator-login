### 2. 🗄️ БАЗЫ ДАННЫХ SUPABASE

#### Создано 3 новых миграции:

##### A. **`20250117_students_management_system.sql`** (Система учеников)

**Таблицы:**

1. **`student_profiles`** - Профили учеников
   ```sql
   - id (UUID, связь с auth.users)
   - full_name (TEXT)
   - email (TEXT, уникальный)
   - phone (TEXT)
   - avatar_url (TEXT)
   - is_active (BOOLEAN)
   - email_verified (BOOLEAN)
   - invited_by (UUID)
   - last_login_at (TIMESTAMP)
   - notification_preferences (JSONB)
   ```

2. **`student_invitations`** - Приглашения с уникальными ссылками
   ```sql
   - id (UUID)
   - email (TEXT)
   - full_name (TEXT)
   - invitation_token (TEXT, уникальный)
   - temp_password_hash (TEXT)
   - temp_password_plain (TEXT) -- для email
   - status ('pending', 'accepted', 'expired')
   - expires_at (TIMESTAMP, +7 дней)
   ```

3. **`password_change_history`** - История смены паролей
   ```sql
   - id (UUID)
   - user_id (UUID)
   - changed_by (UUID) -- кто изменил
   - reason (TEXT) -- 'admin_reset', 'user_change', 'forgot_password'
   ```

**Функции:**

1. **`create_student_invitation()`** - Создать приглашение
   - Генерирует уникальный токен (32 символа)
   - Генерирует временный пароль (12 символов)
   - Возвращает ссылку: `https://integratoronai.kz/invite/{token}`
   - Возвращает пароль для отправки в email

2. **`accept_invitation()`** - Принять приглашение
   - Проверяет токен и срок действия
   - Создаёт профиль ученика
   - Активирует аккаунт

**RLS Policies:**
- ✅ Студенты видят только свой профиль
- ✅ Админы видят и управляют всем
- ✅ Приглашения только для админов

## 🎓 AI-НАСТАВНИК: ИНСТРУКЦИИ

### Роль и задачи:

**AI-наставник** - это персональный помощник каждого ученика, который:
- Помогает в обучении и мотивирует
- Даёт карьерные советы
- Проводит обзоры прогресса
- Решает проблемы с пониманием материала

### Как работает:

#### 1. **Триггеры активации:**

AI-наставник активируется когда:
- AI-куратор обнаруживает проблему (`is_problem_detected = true`)
- AI-аналитик видит падение вовлечённости
- Админ создаёт задачу вручную
- Запланированная сессия (раз в неделю)
- Ученик сам запрашивает помощь

#### 2. **Типы сессий:**

- **`learning_support`** - помощь в понимании материала
- **`motivation`** - мотивационная беседа
- **`career_guidance`** - карьерные советы
- **`progress_review`** - обзор достижений за период
- **`problem_solving`** - решение конкретной проблемы

#### 3. **Контекст сессии:**

Наставник получает:
```json
{
  "student_stats": {
    "lessons_completed": 23,
    "streak_days": 7,
    "achievements": [...],
    "recent_activity": [...]
  },
  "ai_curator_insights": {
    "recent_moods": ["frustrated", "confused"],
    "problems_detected": [
      {
        "type": "understanding",
        "lesson_id": "ml-basics-5",
        "timestamp": "..."
      }
    ]
  },
  "analyst_insights": {
    "engagement_trend": "declining",
    "struggling_topics": ["neural networks"],
    "recommended_actions": [...]
  }
}
```

#### 4. **Выход сессии:**

Наставник создаёт:
- **Рекомендации** (общие советы)
- **Action items** (конкретные задачи)
- **Follow-up date** (когда вернуться к ученику)

Пример:
```json
{
  "recommendations": [
    "Пройти дополнительный урок по нейронным сетям",
    "Попрактиковаться с реальными данными"
  ],
  "action_items": [
    {
      "task": "Завершить упражнение 'Построение первой нейросети'",
      "deadline": "2025-11-10",
      "priority": "high"
    }
  ],
  "follow_up_date": "2025-11-14"
}
```

### Промпт для AI-наставника:

```
Ты - AI-наставник платформы onAI Academy.

ТВОЯ РОЛЬ:
- Персональный ментор и тренер для студента
- Помогаешь преодолевать трудности в обучении
- Мотивируешь и вдохновляешь
- Даёшь карьерные советы

ДОСТУПНЫЕ ДАННЫЕ:
- Прогресс ученика (уроки, достижения, стрик)
- Взаимодействия с AI-куратором (настроения, проблемы)
- Аналитика AI-аналитика (паттерны, тренды)

СТИЛЬ ОБЩЕНИЯ:
- Тёплый, поддерживающий, как старший брат/сестра
- Конкретный и практичный
- Мотивирующий, но реалистичный
- Без академического формализма

СТРУКТУРА СЕССИИ:
1. Приветствие и контекст
2. Обсуждение текущей ситуации
3. Анализ проблем/успехов
4. Конкретные рекомендации
5. План действий
6. Мотивация на завершение

ВАЖНО:
- Всегда опирайся на реальные данные
- Давай конкретные задачи, не абстрактные советы
- Отмечай успехи и прогресс
- Помни предыдущие сессии

Сейчас у тебя сессия типа: {session_type}
Контекст ученика: {context_data}
```

## 📊 AI-АНАЛИТИК: ИНСТРУКЦИИ

### Роль и задачи:

**AI-аналитик** - это система мониторинга и анализа, которая:
- Анализирует прогресс учеников
- Оценивает эффективность AI-куратора
- Выявляет паттерны обучения
- Обнаруживает проблемные области
- Даёт рекомендации для улучшения

### Типы анализов:

#### 1. **`student_progress`** - Анализ прогресса ученика

Анализирует за период:
- Завершённые уроки и модули
- Заработанный XP
- Стрик (дни подряд)
- Скорость обучения
- Качество выполнения (баллы)

Выходные данные:
```json
{
  "analysis_results": {
    "progress_rate": "above_average",
    "engagement_level": "high",
    "learning_velocity": 2.5, // уроков в день
    "quality_score": 85, // средний балл
    "trend": "improving"
  },
  "insights": [
    "Ученик показывает стабильный рост",
    "Высокая вовлечённость в последнюю неделю",
    "Отлично справляется с практическими заданиями"
  ],
  "recommendations": [
    "Можно переходить к более сложным темам",
    "Предложить роль наставника для новичков"
  ]
}
```

#### 2. **`ai_curator_effectiveness`** - Эффективность AI-куратора

Анализирует:
- Количество взаимодействий
- Определённые настроения
- Обнаруженные проблемы
- Решённые проблемы
- Время ответа
- Качество ответов (по обратной связи)

Метрики:
```json
{
  "metrics": {
    "total_interactions": 45,
    "problems_detected": 8,
    "problems_resolved": 6,
    "average_response_time_sec": 3.5,
    "positive_feedback_rate": 0.92,
    "mood_detection_accuracy": 0.87
  }
}
```

#### 3. **`learning_patterns`** - Паттерны обучения

Выявляет:
- Предпочтительное время обучения
- Длительность сессий
- Частота пауз
- Сложные темы
- Лёгкие темы
- Стиль обучения

#### 4. **`engagement_analysis`** - Анализ вовлечённости

Отслеживает:
- Частота входов
- Время на платформе
- Взаимодействия с контентом
- Прогресс по курсам
- Социальная активность

#### 5. **`problem_areas`** - Проблемные области

Обнаруживает:
- Темы где застревают
- Частые ошибки
- Негативные настроения
- Падение мотивации
- Технические проблемы

#### 6. **`overall_platform`** - Общая аналитика платформы

Анализ всей платформы:
- Общее количество активных учеников
- Средний прогресс
- Самые популярные курсы
- Проблемные модули
- Эффективность AI-систем

### Как работает:

#### Автоматические отчёты:

1. **Ежедневно (00:00):**
   - Обзор активности за день
   - Новые алерты и проблемы
   - Топ учеников дня

2. **Еженедельно (понедельник 09:00):**
   - Детальный анализ прогресса каждого ученика
   - Сравнение с предыдущей неделей
   - Рекомендации для AI-наставника

3. **Ежемесячно (1-е число 10:00):**
   - Полный отчёт по платформе
   - Тренды и паттерны
   - Стратегические рекомендации

#### Триггерные отчёты:

- Падение вовлечённости ученика (>3 дня без входа)
- Негативное настроение 3+ раза подряд
- Застревание на уроке >7 дней
- Низкий балл за тест (<50%)

### Промпт для AI-аналитика:

```

ТВОЯ РОЛЬ:

ДОСТУПНЫЕ ДАННЫЕ:

ТИПЫ АНАЛИЗА:

ВАЖНО:
```

## 📞 ВОПРОСЫ?

### "Как создать первого админа?"

```sql
-- В Supabase SQL Editor:
UPDATE auth.users 
SET raw_user_meta_data = jsonb_set(
  raw_user_meta_data, 
  '{role}', 
  '"admin"'
)
WHERE email = 'твой@email.com';
```

### "Как протестировать добавление ученика без email?"

Используй функцию напрямую:
```sql
SELECT * FROM create_student_invitation(
  'test@student.com',
  'Тест Тестов'
);
```

Вернёт токен и пароль для ручной отправки.

### "Что если я хочу изменить срок приглашения?"

В миграции меняй:
```sql
expires_at TIMESTAMP WITH TIME ZONE DEFAULT NOW() + INTERVAL '7 days'
```

На:
```sql
expires_at TIMESTAMP WITH TIME ZONE DEFAULT NOW() + INTERVAL '30 days'
```
