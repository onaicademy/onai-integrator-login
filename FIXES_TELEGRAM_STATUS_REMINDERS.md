### 3. 📅 Scheduler для напоминаний (FIXED!)
**Проблема:** 
```
❌ [Scheduler] Error: "Could not find a relationship between 'goals' and 'users'"
```

**Решение:**
- ✅ Убрал сложный JOIN через `users!inner()`
- ✅ Теперь сначала загружаются задачи, затем отдельно данные пользователей
- ✅ Проверяется `telegram_connected = true` перед отправкой

**Как работает:**
1. Каждую минуту Scheduler проверяет задачи с:
   - `telegram_reminder = true`
   - `status != 'done'`
   - `due_date` в ближайшие 60 минут
2. Для каждой задачи загружает данные пользователя
3. Вычисляет: `dueDate - reminder_before` = время отправки
4. Если время пришло - отправляет напоминание в Telegram

**Файлы:**
- `backend/src/services/reminderScheduler.ts` (строки 39-79)

### 4. 🧪 Тестовый endpoint для напоминаний (NEW!)
**Что добавлено:**
- ✅ Новый API endpoint: `POST /api/telegram/test-reminder`
- ✅ Запускает проверку напоминаний вручную (не нужно ждать 1 минуту)

**Как протестировать:**

#### Шаг 1: Создай задачу с напоминанием
1. Открой Kanban (http://localhost:8080/neurohub)
2. Создай задачу:
   - **Название:** "Тестовая задача для напоминания"
   - **Дата:** Сегодня
   - **Время:** Через 5 минут от текущего времени
   - **Telegram:** Подключи бота (если еще не подключен)
   - **Напомнить за:** 15 минут

#### Шаг 2: Измени время на ближайшее
1. Открой задачу снова
2. Измени время на **ПРОШЕДШЕЕ** (например, 10 минут назад)
3. Так мы симулируем "задача должна была быть 10 минут назад"
4. Напоминание "за 15 минут" должно сработать

#### Шаг 3: Запусти тестовый триггер
Открой новый терминал и выполни:

```bash
curl -X POST http://localhost:3000/api/telegram/test-reminder
```

**Ожидаемый результат:**
```json
{
  "success": true,
  "message": "Reminder check triggered! Check console and Telegram."
}
```

#### Шаг 4: Проверь Telegram
В течение 1-2 секунд тебе придет сообщение:
```
⚡ Напоминание о задаче!

👤 Saint
📋 Тестовая задача для напоминания

⏰ Осталось 5 минут
📅 Дедлайн: 24 ноября в 22:45

💪 Самое время завершить задачу!
```
