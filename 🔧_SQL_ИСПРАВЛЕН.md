## ✅ ЧТО ИСПРАВЛЕНО:

### **Проблема:**
```
ERROR: 42P10: there is no unique or exclusion constraint matching the ON CONFLICT specification
```

### **Причина:**
- `auth.users` не имеет уникального ключа по `email` для `ON CONFLICT (email)`
- Нельзя использовать `ON CONFLICT` без уникального индекса

### **Решение:**
1. ✅ Убрал `ON CONFLICT (email)`
2. ✅ Добавил `DELETE FROM auth.users WHERE email = '...'` перед INSERT
3. ✅ Упростил логику - сначала удаляем, потом создаём заново
