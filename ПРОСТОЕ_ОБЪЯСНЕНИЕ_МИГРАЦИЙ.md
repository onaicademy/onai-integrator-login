## 🏗️ ЧТО ТАКОЕ МИГРАЦИИ

**Миграции = SQL файлы**, которые **создают таблицы** в базе данных.

У нас есть **5 файлов** (в папке `supabase/migrations/`):
1. Первые 2 - базовые таблицы
2. Таблицы для AI-куратора
3. Таблицы для достижений

**Эти файлы СУЩЕСТВУЮТ в коде, НО ЕЩЁ НЕ ВЫПОЛНЕНЫ в Supabase!**

---

## ❌ ПРОБЛЕМА СЕЙЧАС

База данных Supabase выглядит так:
```
┌─────────────┐
│  SUPABASE   │
│             │ ← ПУСТО!
│  (no tables)│
└─────────────┘
```

А должна быть так:
```
┌─────────────────────────┐
│  SUPABASE               │
│                         │
│  ✅ user_achievements    │
│  ✅ user_statistics      │
│  ✅ ai_curator_threads   │
│  ✅ ai_curator_messages  │
│  ✅ ...ещё 3 таблицы     │
└─────────────────────────┘
```

## ✅ РЕШЕНИЕ

**ТЕБЕ нужно:**

### Вариант 1 (САМЫЙ ПРОСТОЙ - 5 минут):

1. Открой: https://supabase.com/dashboard/project/capdjvokjdivxjfddmx
2. Слева найди **"SQL Editor"**
3. Нажми **"New query"**
4. Скопируй каждый файл миграции и нажми **"Run"**

**Как скопировать?** В терминале:
```bash
cat "supabase/migrations/20250115_add_ai_curator_chat_tables.sql"
```
Вывод → Копируй → Вставь в SQL Editor → Run

**Повтори для всех 5 файлов.**
