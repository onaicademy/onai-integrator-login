## 🏗️ **1. ОБЩАЯ АРХИТЕКТУРА**

### **Stack:**
- **Frontend:** React 18 + TypeScript + Vite + Tailwind CSS + Framer Motion
- **Backend:** Node.js + Express.js + TypeScript
- **Database:** Supabase (PostgreSQL)
- **Storage:** Supabase Storage (файлы), Cloudflare R2 (видео)
- **AI:** OpenAI (GPT-4, Whisper, Vision API, Assistants API)
- **Deployment:** 
  - Frontend → Vercel (https://onai.academy)
  - Backend → DigitalOcean (https://api.onai.academy)

## 📈 **3. AI-АНАЛИТИК - Работа с метриками**

### **Местоположение:** `/admin/ai-analytics`

### **Функционал:**

#### ✅ **РЕАЛИЗОВАНО:**

1. **📊 Детектор проблемных студентов:**
   - Система оценки по 5 критериям:
     - Активность (последний визит, частота логинов)
     - Настроение (средний mood, тренд)
     - Прогресс (уроки за неделю, completion rate)
     - Взаимодействие с AI (количество вопросов)
     - Результаты тестов (средний балл, попытки)
   - Уровни внимания:
     - 🔴 **Critical** (0-40 баллов) - требуется срочное вмешательство
     - 🟠 **Warning** (41-60) - нужна поддержка
     - 🟡 **Attention** (61-75) - следить за прогрессом
     - 🟢 **Normal** (76-85) - всё хорошо
     - 💚 **Excellent** (86-100) - отличный прогресс

2. **🚩 Красные флаги:**
   - Не заходил >7 дней
   - Настроение <3/5
   - Падение настроения >1 балл/неделю
   - 0 уроков за 7 дней
   - Застрял на уроке >3 дней
   - Низкий балл тестов (<60%)
   - Много попыток тестов (>3)

3. **⚠️ Жёлтые флаги:**
   - Не заходил 3-7 дней
   - Настроение 3-4/5
   - Мало вопросов AI (<3 за неделю)
   - Низкий completion rate (<50%)

4. **💡 Рекомендации для кураторов:**
   - Автоматическая генерация действий на основе флагов
   - Приоритизация студентов
   - Детальная статистика по каждому студенту

5. **🤖 Детектор конфликтов AI:**
   - Отслеживание проблемных ответов бота
   - Типы конфликтов:
     - INCORRECT_ANSWER (неверный ответ)
     - HALLUCINATION (галлюцинация)
     - MISUNDERSTOOD_QUESTION (непонимание вопроса)
     - REPETITIVE_ANSWER (повторяющийся ответ)
     - INAPPROPRIATE_TONE (неподходящий тон)
     - TECHNICAL_ERROR (техническая ошибка)
     - INCOMPLETE_ANSWER (неполный ответ)
   - Статусы: pending, reviewed, resolved, false_positive
   - Детальная информация: студент, вопрос, ответ, confidence score

#### ❌ **НЕ РЕАЛИЗОВАНО:**

1. **Интеграция с NeuroHub:**
   - AI-Наставник НЕ имеет доступа к данным AI-Аналитика
   - Нельзя спросить у AI: "Как у меня дела по сравнению с другими?"
   - Нет персонализированных рекомендаций на основе аналитики

2. **Автоматизация:**
   - Нет автоматических уведомлений кураторам о проблемных студентах
   - Нет email/Telegram интеграции
   - Нет автоматического создания задач для кураторов

3. **Advanced Analytics:**
   - Нет прогнозирования (predictive analytics)
   - Нет кластеризации студентов по типам обучения
   - Нет A/B тестирования разных подходов

4. **Экспорт данных:**
   - Нет экспорта в Excel/CSV
   - Нет PDF отчётов
   - Нет API для внешних систем

## 🚨 **6. ТЕКУЩИЕ ПРОБЛЕМЫ**

### **Критические:**

#### 1. **Кнопка скрепки не реагирует (PDF attachment)**

**Описание:**
- Пользователь жалуется: "кнопка скрепки практически не реагирует"
- PDF-файлы не прикрепляются или не показываются

**Анализ кода:**

```typescript
// src/pages/NeuroHub.tsx

// ✅ Функция обработки файлов ЕСТЬ:
const handleFileUpload = (e: React.ChangeEvent<HTMLInputElement>) => {
  const files = Array.from(e.target.files || []);
  setAttachedFiles(prev => [...prev, ...files]);
};

// ✅ Input настроен правильно:
<input
  ref={fileInputRef}
  type="file"
  multiple
  className="hidden"
  onChange={handleFileUpload}
  accept="image/*,.pdf,.doc,.docx,.txt"
/>

// ✅ Кнопка скрепки:
<Button
  size="icon"
  variant="outline"
  onClick={() => fileInputRef.current?.click()}
  disabled={isSending}
  className="border-[#00ff00]/30 hover:bg-[#00ff00]/10 text-[#00ff00]"
>
  <Paperclip className="w-5 h-5" />
</Button>
```

**Возможные причины:**
1. **`isSending` застрял в `true`** → кнопка disabled
2. **Файлы не показываются после прикрепления** → пользователь думает что не работает
3. **Ошибка на Backend при обработке PDF** → файл отправляется, но ломается

**Проверка:**
- Нужно добавить визуальное отображение прикреплённых файлов
- Нужно добавить логи в консоль при клике на скрепку

#### 2. **PDF-скан не обрабатывается**

**Описание:**
- AI отвечает: "Похоже, что загруженный вами PDF-файл содержит изображения, из которых невозможно извлечь текст напрямую."
- Пользователь пишет: "твой pdf фарсер не работает"

**Причина:**
- `pdf-parse` извлекает только **текстовый слой** PDF
- Если PDF — это скан (изображение), то `extractedText` будет пустым
- Backend возвращает пустой текст → AI думает что файла нет

**Код проблемы:**

```typescript
// backend/src/services/fileProcessingService.ts

export async function extractTextFromPDF(buffer: Buffer): Promise<string> {
  try {
    const data = await pdfParse(buffer);
    console.log(`[PDF] Извлечено ${data.text.length} символов`);
    
    if (!data.text || data.text.trim().length === 0) {
      console.warn('⚠️ PDF не содержит текстового слоя (возможно скан)');
      return ''; // ❌ ПРОБЛЕМА: Возвращаем пустую строку
    }
    
    return data.text;
  } catch (error) {
    console.error('[PDF] Ошибка парсинга:', error);
    throw error;
  }
}
```

**Решение:**
1. Детектировать пустой текст на Backend
2. Если `extractedText === ''`:
   - **Вариант А:** Конвертировать PDF → изображение → Vision API
   - **Вариант Б:** Использовать OCR (`tesseract.js`)
   - **Вариант В:** Вернуть специальное сообщение для AI

#### 3. **Прикреплённые файлы не показываются в чате**

**Описание:**
- Пользователь: "у меня опять не работает функция связано с pdf файлом... в окне не показывает какой именно файл я подгрузил пред лад вот это те же нету"

**Проблема:**
- Файлы прикрепляются, но нет preview
- В сообщениях чата не показывается информация о файлах

**Нужно добавить:**
