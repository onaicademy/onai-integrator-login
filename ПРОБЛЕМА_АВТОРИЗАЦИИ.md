### 2. Что происходит:

1. **При сборке (build):**
   - Vite читает `.env`
   - Заменяет `import.meta.env.VITE_*` на реальные значения
   - Создаёт production bundle

2. **При загрузке страницы:**
   - Supabase клиент инициализируется
   - Логирование показывает что key есть (keyLength: 208)

3. **При попытке входа:**
   - Вызывается `supabase.auth.signInWithPassword()`
   - Запрос отправляется к Supabase
   - **API key НЕ включён в headers запроса**
   - Supabase возвращает 400 Error

### 3. Возможные причины:

#### **Причина #1: Vite не подхватывает .env при production build**
- Переменные должны начинаться с `VITE_`
- Файл `.env` должен быть в корне проекта
- При сборке переменные "зашиваются" в код

#### **Причина #2: Supabase client неправильно инициализирован**
- Возможно нужны дополнительные опции
- Возможно проблема с версией `@supabase/supabase-js`

#### **Причина #3: API key имеет неправильный формат**
- Key должен быть JWT токеном (начинается с `eyJ...`)
- Длина ~200+ символов

#### **Причина #4: Кеш браузера**
- Старый билд без API key кешируется
- Нужна жёсткая перезагрузка
