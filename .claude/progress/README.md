# Progress Reports

This directory contains session progress reports. Each file documents changes made during a Claude Code session.

## File Naming Convention

```
YYYY-MM-DD_session.md        # Daily session report
YYYY-MM-DD_session_N.md      # Multiple sessions in one day (N = 2, 3, ...)
```

## Report Structure

Each report should include:

1. **Session Info** - Date, time, duration
2. **Summary** - Brief overview of what was accomplished
3. **Changes Made** - List of files created/modified
4. **Issues Fixed** - Bugs resolved
5. **Feedback** - User feedback received
6. **Next Steps** - Pending tasks for future sessions

## Template

```markdown
# Progress Report - YYYY-MM-DD

## Session Info
- **Date**: YYYY-MM-DD
- **Time**: HH:MM - HH:MM (Timezone)
- **Duration**: Xh Xm

## Summary
Brief 1-2 sentence overview.

## Changes Made

### New Files
- `path/to/file.ts` - Description

### Modified Files
- `path/to/file.ts` - What changed

### Deleted Files
- `path/to/file.ts` - Why deleted

## Issues Fixed
- [FIX] Description of fix (commit hash)

## Commits
- `abc1234` - Commit message

## Feedback Received
- User requested: ...
- User noted: ...

## Next Steps
- [ ] Task 1
- [ ] Task 2
```
