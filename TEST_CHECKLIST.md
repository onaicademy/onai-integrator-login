# Чеклист для проверки исправлений

## 1. Проверка таблицы profiles в Supabase
- [ ] Открой https://supabase.com/dashboard/project/arqhkacellqbhjhbebfh/editor
- [ ] Проверь, существует ли таблица `profiles`
- [ ] Проверь, есть ли в ней колонка `role`
- [ ] Проверь, есть ли в ней колонка `is_active`

## 2. Проверка авторизации (без таймаута)
- [ ] Открой https://onai.academy/login
- [ ] Открой DevTools (F12) → Console
- [ ] Войди с тестовыми данными
- [ ] Проверь логи:
  - `🔍 Проверка авторизации...`
  - `✅ Сессия найдена: <email>`
  - `🔄 Редирект на: /courses`

## 3. Проверка загрузки роли (MainLayout)
- [ ] После входа открой Console
- [ ] Проверь логи:
  - `🔍 MainLayout: Начало загрузки роли...`
  - `👤 User ID: <uuid>`
  - `📧 Email: <email>`
  - `✅ Профиль найден, роль: student` (или admin)

## 4. Проверка фильтрации меню (Sidebar)
### Для студента:
- [ ] Войди как студент
- [ ] Проверь Console:
  - `📋 AppSidebar роль: student`
  - `🚫 Скрываем админ-панель для студента`
  - `✅ Показываем пунктов: 6`
- [ ] Проверь, что НЕТ пункта "Админ панель" в меню

### Для админа:
- [ ] Войди как admin (saint@onaiacademy.kz)
- [ ] Проверь Console:
  - `📋 AppSidebar роль: admin`
  - `✅ Показываем пунктов: 7`
- [ ] Проверь, что ЕСТЬ админское меню с Dashboard

## 5. Проверка StudentsActivity
- [ ] Войди как admin
- [ ] Открой /admin/students
- [ ] Проверь Console:
  - `📋 Загрузка студентов...`
  - `✅ Загружено студентов: <число>`
- [ ] Проверь, что список студентов отображается
- [ ] Проверь кнопки управления (смена роли, деактивация)

## 6. Проверка на ошибки
- [ ] Открой Console (F12)
- [ ] Проверь, нет ли ошибок красным цветом
- [ ] Проверь Network → проверь статусы запросов к Supabase

## ❌ Если что-то не работает:
1. Сделай скриншот ошибки из Console
2. Скопируй текст ошибки
3. Проверь Network → найди failed запросы
4. Отправь мне информацию для исправления
